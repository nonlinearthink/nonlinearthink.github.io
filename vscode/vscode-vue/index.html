VSCode Vue 运行环境配置

发表于2020-05-16|更新于2021-03-11|VSCode 配置手册
|字数总计:884|阅读时长:3分钟|阅读量:

Vue代码 可能出现在两种文件中:

  • 一种是面向初学者的，在 html 代码直接嵌入。
  • 另一种是工程化项目中使用的，以 vue 为后缀的文件。

HTML 文件中代码补全

这可能对一些新手比较有用，之后熟悉了 Vue工程化 之后基本没用了。

我一开始安装了一个 VueHelper 插件来解决问题。

不过，现在我不再推荐使用 VueHelpr 了。因为插件 Vue 2 Snippets 比它更加强大，主要体现在这两个方面:

  • 补全的指令更多。
  • 指令还可以支持 vue 和 jade 文件的补全，而前者仅仅支持 js、ts、html。

直接在插件仓库中搜索 hollowtree.vue-snippets ，安装即可。

Mark一下，Vue 3 出了，应该很快会有 Vue 3 的代码补全，待更新。

Vue 文件中的代码补全

这方面 Vetur 已经做的很好了。

在插件仓库中搜索 octref.vetur ，安装即可。

使用快捷键 Ctrl+Shift+P 打开 settings.json 。

配置 Vue 格式化

通过 editor.defaultFormatter ，配置默认格式化插件。

通过 editor.formatOnSave ，支持保存的时候自动格式化。

通过 editor.formatOnPaste ，支持粘贴的时候自动格式化。

配置 vetur

上一步配置了 Vue文件 的默认格式化插件为 Vetur 。现在，我们需要配置一下 Vetur，来支持 vue 的各种补全。

这里需要用到另外一个插件，prettier 是一款非常强大的前端格式化插件，在插件仓库搜索 esbenp.prettier-vscode，安装即可。

我们设置上面提到的所有的东西的默认格式化插件为 prettier。

我们只配置了 scss 和 less，如果你使用的是其他的 css预处理器，只需要修改一个后缀即可。

经过了这样的配置之后，在选择 Vue-cli 初始化的时候，Linter / Formatter 那一项应该使用 ESLint + Prettier。
