盒模型(box model)

了解盒模型的概念

CSS 的盒模型并不是什么很难理解的东西，你只需要记住它由四部分构成：content(内容)、padding(内边距)、border(边框)、margin(外边距)。

举一个例子来说明:

从图中左下角有一副盒模型的解刨图，从图中我们可以看到：

  • content 的大小是 $400\times250$

    这和 width: 50vw; 和 height: 50vh; 有关，400px 和 250px 是两者的计算值。

  • 四边的 padding 大小都是 32px

    padding: 2em;，在CSS: 单位计算与响应式布局中我提过，浏览器默认的字号是 16px，所以 2em 的计算值是 32px。

  • 有 1px 的 border

    border: 1px solid black;，分别代表边框的粗细、类型、颜色。

  • margin 的上下大小是 125px；左右大小是 151.5px。

    margin: 25vh auto;，代表左右的 margin 的值是 25vh(计算值为 151.5px)，上下的 margin 值均分，也就是垂直方向会变成居中。

    这是一种常见的让元素居中的写法，但是只能让一个方向居中，你无法写两个 auto。

对照我们的 css 代码，你就能理解 盒模型 的概念了。

盒模型时钟

简写属性

padding、border、margin 这三个其实是简写属性，所谓的简写属性，是指在一个属性中有超过两个的值。例如，padding 和 margin 的 具体属性:

|padding|margin|
|-|-|-|
|padding-top|margin-top|
|padding-right|margin-right|
|padding-bottom|margin-bottom|
|padding-left|margin-left|

时钟顺序

盒模型的属性值，只要涉及到方向，全都是顺时钟顺序，上、右、下、左。

但是你肯定看到过定义 padding 和 margin 的值不是4个，而是2个，或者1个，甚至3个。

这怎么回事呢？

对称原则

以 margin 为例。

  • 一个值: margin: 1em;

    这种情况下，代表上、右、下、左四个方向的值全都相同。

  • 两个值: margin: 1em .8em;

    这种情况下，只定义了两个方向，根据时钟顺序，分别是 上方 和 右方。
    接下来介绍的对称原则是指，假如 下方 和 左方 的值没有定义，则使用对称的 上方 和 右方 的值。

  • 三个值: margin: 1em .8em 2em;

    这种情况下，根据时钟顺序，定义了三个值，分别是 上方、右方、下方。
    根据对称原则，没有定义的 左方 使用对称方(右方) 的值。

只要记住了 时钟顺序 和 对称原则，你就基本上掌握了 盒模型。

边框(border)

border 也是一个简写属性:

border 可取值
border-width 所有长度单位 | thin | medium | thick
border-style none | hidden | dotted | dashed | solid | double | groove | ridge | inset | outset
border-color 所有颜色的表示方式

border 的一些实例

border: 0.2em dotted black;

border: 0.2em dashed red;

border: 0.5em solid black;

border: 0.5em double black;

盒模型 box-sizing

box-sizing属性

默认情况下，box-sizing 的值是 content-box，意味着 hight 和 width 两个值作用的对象是 content。

在某些情况下，这会给我们布局带来一些麻烦，因为如果存在 padding、border、margin，那么我们的布局可能会被打乱。

举个例子，现在我们要在为了在同一行放两个元素 main 和 aside，我们设置 main 的宽度为 70%，aside 的宽度为 30%。

为什么 main 和 aside 不在同一行?
我们之前说到 box-sizing 的默认值是 content-box。也就是说 width 的百分比约束仅仅只针对盒模型中的 content 部分，但我们这里设置了 padding 和 border，所以元素的实际长度和 width 不符。

为了解决这个问题，我们稍微改一下 style.css，把 box-sizing 的值修改为 border-box，这样就把 padding、border 也放入了 width 的宽度约束中:

间隔的原理很简单，就是在原来 30% 的 width 基础上减去 0.5em，同时为 aside 添加 0.5em 的左外边距。

很多 CSS 新手总是没有意识去精确计算 CSS 的布局，而是使用 魔术数值，比如上题中，也可以这么做，把 width 设置成 29%，margin-left 设置成 0.5em。

这的确在一些情况下是可取的，但是 29% 是不稳定的值，会随着屏幕的调整而变化，假如屏幕窄到一定程度，就会出现布局混乱的情况，这个 29% 就叫 魔术数值 ，意思是它总是变化的，不满足响应式布局的要求。

我们必须做到一丝不苟地去精确计算 CSS 布局。

猫头鹰选择器

上面的方法对于有些情况可能不是很适合。

现在，我们想要仅在元素的中间添加间隙，而不在最前面和最后面添加间隙。这可以怎么做呢？

一种很老套的方法就是使用 :not(:first-child) 和 :not(:last-child) 去除两边的样式。

有一种非常巧妙的方法，叫做猫头鹰选择器。因 * + * 长得很像猫头鹰而得名。

在此之前我们先来回顾一下兄弟选择器。

+ 是当元素拥有这个兄弟元素的时候触发的选择器。

* + * 的意思是，当两个元素是兄弟元素的关系的时候，在后面的那个元素上添加新的 CSS 规则。
